# Zapp Product Definition

## Vision

Zapp is a personal finance planning app for desktop and mobile. It helps the user plan annual expenses, distribute income, track month-by-month reality, and make better decisions before committing to large purchases.

## Primary User

The first user is the owner of the app. The product is optimized for one person's real financial workflow first, while keeping the design clean enough to scale to more users later.

## Core Problem

The current planning method works partially but is disorganized. Financial information is spread across different places, calculations are manual, and decisions require checking many moving parts. Zapp should turn that into a structured, repeatable method.

## Product Goals

- Build an annual budget from income and expense assumptions.
- Track actual monthly execution against the plan.
- Show how much can be spent, saved, or committed each month.
- Help distribute salary, bonuses, and extra income intentionally.
- Simulate the impact of large purchases before making them.
- Automate repetitive data capture where it is reliable and worth the complexity.

## Non Goals For The First Version

- It is not a bank replacement.
- It is not a generic accounting system.
- It is not a multi-user SaaS at launch.
- It should not start with complex automation before the manual workflow is solid.

## Key Decisions Zapp Should Support

- Can I make this purchase now?
- How much should I spend this month?
- How should I distribute my income?
- Which months are financially tight?
- Am I spending more than planned?
- How much can I safely save?

## Product Principles

- The annual plan is the source of truth.
- Monthly tracking should be fast and low friction.
- Every automation must be explainable and reversible.
- Manual entry must always exist as a fallback.
- The dashboard should answer decisions, not just show charts.
